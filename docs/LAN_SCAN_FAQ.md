# 局域网扫描说明

## 重要提示 ⚠️

**局域网扫描无法发现同一台电脑的多个浏览器窗口，这不是 Bug！**

## 为什么同一台电脑扫描不到？

### 技术原因

1. **mDNS 的工作机制**
   - mDNS 通过网络层广播实现设备发现
   - 每台设备有唯一的 IP 地址
   - 同一台电脑的所有进程共享同一个 IP

2. **服务器端绑定**
   ```go
   // 每个 AirLink 实例启动一个 mDNS 服务
   // 但它们都绑定在同一个 IP 地址上
   server, err := mdns.NewServer(&mdns.Config{Zone: service})
   ```

3. **网络栈限制**
   - 操作系统的网络栈是共享的
   - UDP 端口 5353（mDNS）只能被一个进程监听
   - 即使启动多个 AirLink，也是同一台设备

### 类比说明

想象你在一个房间里：
- **局域网扫描** = 大声喊"有人吗？"
- **不同设备** = 不同房间的人会回应
- **同一电脑的多个窗口** = 自己对着镜子喊，只能听到自己的回声

## 正确的测试方法

### ✅ 可以扫描到

**物理设备之间：**
```
设备 A（电脑）        设备 B（手机）
   ↓                      ↓
   WiFi 路由器
   ↓
 mDNS 广播
   ↓
两设备可以互相发现
```

**测试步骤：**
1. 准备两台不同的物理设备（手机、平板、另一台电脑等）
2. 连接到同一 WiFi 网络
3. 两台设备都打开 AirLink
4. 点击"扫描局域网"
5. 应该能看到对方设备

### ❌ 无法扫描到

**同一台电脑：**
```
同一台电脑
├── 浏览器窗口 1 (AirLink)
├── 浏览器窗口 2 (AirLink)
└── 隐私窗口 (AirLink)

↓ (共享同一个 IP)
网络接口
```

这些都无法互相发现，因为：
- 共享同一个 IP 地址
- 共享同一个网络栈
- mDNS 无法区分

**无法扫描的场景：**
- ❌ 同一台电脑的不同浏览器窗口
- ❌ 同一台电脑的普通窗口和隐私窗口
- ❌ 同一台电脑的不同浏览器（Chrome vs Firefox）
- ❌ 虚拟机（默认 NAT 网络模式）

## 解决方案：使用 PIN 码

### PIN 码连接（推荐）

**优点：**
- ✅ 可以在同一台电脑测试
- ✅ 可以跨网络、跨地域连接
- ✅ 安全可靠
- ✅ 简单易用（6位数字）

**使用方法：**
```
1. 窗口 A：创建房间 → 获得 PIN（如 123456）
2. 窗口 B：输入 PIN → 加入房间
3. 连接建立，开始传输
```

### 局域网扫描（辅助功能）

**用途：**
- 方便局域网内快速连接
- 无需手动输入 PIN
- 更好的用户体验

**限制：**
- 只能发现不同的物理设备
- 受限于网络环境（防火墙、网络隔离）
- 不是核心功能

## API 响应说明

### 正常响应（空列表）

当在同一台电脑测试时：

**请求：**
```
GET http://localhost:8080/api/scan
```

**响应：**
```json
{
  "devices": []
}
```

**这是正常的！** 因为同一台电脑无法发现自己的其他窗口。

**响应：**
```json
{
  "devices": [

      "device_id": "uuid-xxx",

      "ip": "192.168.1.100"
    },
    {
      "device_id": "uuid-yyy",

      "ip": "192.168.1.101"
    }
  ]
}
```

## 常见问题

### Q: 为什么我在同一台电脑上扫描不到？
**A:** 这是正常的！同一台电脑的多个窗口共享同一个 IP，mDNS 无法区分。请使用 PIN 码连接。

### Q: 在不同电脑上也扫描不到怎么办？
**A:** 可能的原因：
1. 防火墙阻止了 UDP 5353 端口
2. 网络隔离（企业网络、公共 WiFi）
3. 不在同一子网
4. 路由器禁用了多播

**解决方案：** 使用 PIN 码连接（可以跨网络）

### Q: 局域网扫描有什么用？
**A:** 
- 在支持的网络环境下提供便利
- 无需手动输入 PIN
- 更直观的设备列表
- 但**不是必需功能**，PIN 码才是核心

### Q: 我该用哪种方式连接？
**A:**
| 场景 | 推荐方式 |
|------|----------|
| 同一台电脑测试 | PIN 码 |
| 局域网内不同设备 | 局域网扫描 或 PIN 码 |
| 不同网络的设备 | PIN 码 |
| 防火墙严格的网络 | PIN 码 |

**结论：PIN 码是万能的，局域网扫描是锦上添花。**

## 技术细节

### mDNS 工作流程

```
1. 服务器启动
   ├── 绑定 UDP 5353
   ├── 加入多播组 224.0.0.251
   └── 广播服务信息

2. 客户端扫描
   ├── 发送查询请求（_airlink._tcp.local）
   ├── 等待响应（2秒）
   └── 收集响应设备

3. 同一台电脑的问题
   ├── 只有一个 IP 地址
   ├── 多个进程共享网络栈
   └── 无法区分"自己"和"其他窗口"
```

### HTTP 探测作为备用

如果 mDNS 失败，会尝试 HTTP 探测：

```go
// 扫描 192.168.x.x 网段的所有 IP
for ip := range subnet {
    response := httpGet(ip, port, "/api/ping")
    if response.OK {
        devices.append(response.DeviceInfo)
    }
}
```

**但同样无法发现同一台电脑的其他窗口。**
